# ⚡ Быстрый старт 8800.life

## Проблема которую мы решаем

У тебя было 2 отдельных сервера:
- Бот на одном сервере
- Прокси на другом (104.233.9.112)

Клиенты не могли подключиться, потому что домен 8800.life не резолвился к IP прокси-сервера.

## Решение

Теперь всё работает на ОДНОМ сервере в Docker контейнерах.

## 🚀 Установка за 3 шага

### Шаг 1: Подготовка

```bash
# Подключиться к серверу
ssh root@104.233.9.112

# Перейти в директорию проекта
cd /root/Proxy8800

# Сделать скрипты исполняемыми
chmod +x deploy.sh check-config.sh
```

### Шаг 2: Настройка .env

```bash
nano .env
```

Измени эти строки:
```env
# Твой IP сервера (ВАЖНО!)
PROXY_DOMAIN=104.233.9.112

# Токен бота от @BotFather
BOT_TOKEN=8331254337:AAEzG59GjJ-Ysu1eXinSBV6cEq8b-67N9EU

# Секрет для MTProto (можно оставить как есть)
MTPROTO_SECRET=dd2ae5891b2b9b9b811b212050843193aa

# API ключ (измени на свой случайный)
API_KEY=твой-секретный-ключ-123456
```

Сохрани: `Ctrl+X`, `Y`, `Enter`

### Шаг 3: Запуск

```bash
# Запустить всё
./deploy.sh
```

Скрипт автоматически:
- Проверит Docker
- Соберет контейнеры
- Запустит бота и прокси
- Покажет ссылку для подключения

## ✅ Проверка

```bash
# Проверить конфигурацию
./check-config.sh

# Посмотреть логи
make logs

# Статус сервисов
make status
```

## 📱 Подключение к прокси

После запуска получишь ссылку вида:
```
https://t.me/proxy?server=104.233.9.112&port=8800&secret=dd2ae5891b2b9b9b811b212050843193aa
```

Открой её в Telegram → "Подключить прокси"

## 🔧 Управление

```bash
make logs       # Все логи
make logs-bot   # Только бот
make logs-proxy # Только прокси
make restart    # Перезапустить
make down       # Остановить
make up         # Запустить
```

## 🐛 Если что-то не работает

### Прокси не подключается

```bash
# Проверь порт
netstat -tulpn | grep 8800

# Проверь логи
make logs-proxy

# Проверь IP в .env
./check-config.sh
```

### Бот не отвечает

```bash
# Проверь токен в .env
grep BOT_TOKEN .env

# Проверь логи
make logs-bot

# Перезапусти
make restart
```

### Контейнеры не запускаются

```bash
# Пересоздай всё
make down
docker system prune -f
make build
make up
```

## 📊 Мониторинг

